# Business Rules - Backend Core Order

## BR-AUTH: 인증 규칙

### BR-AUTH-01: 관리자 로그인
- 매장 코드 + 사용자명 + 비밀번호로 인증
- bcrypt로 비밀번호 검증
- 성공 시 JWT 토큰 발급 (만료: 16시간)
- 매장당 관리자 1명 (메인 포스기)

### BR-AUTH-02: 로그인 시도 제한
- 연속 5회 실패 시 계정 잠금 (15분)
- 잠금 해제 후 시도 횟수 리셋
- 성공 시 시도 횟수 리셋

### BR-AUTH-03: 룸 태블릿 로그인
- 매장 코드 + 룸 번호 + 비밀번호로 인증
- 성공 시 JWT 토큰 발급 (만료: 16시간)
- 로그인 정보 클라이언트 로컬 저장 → 자동 로그인

## BR-ORDER: 주문 규칙

### BR-ORDER-01: 주문 생성
- 활성 룸 세션이 있어야 주문 가능
- 세션이 없으면 첫 주문 시 자동 세션 생성 (2시간)
- 주문 번호: 매장코드-날짜-순번 형식 (예: HP-20260317-001)
- 총 금액 = Σ(단가 × 수량)
- 주문 생성 시 SSE로 관리자에게 실시간 알림

### BR-ORDER-02: 주문 상태 변경
- 순차적: PENDING → PREPARING → COMPLETED
- 되돌리기 불가
- 상태 변경 시 SSE로 관리자에게 알림

### BR-ORDER-03: 주문 삭제 (관리자)
- 관리자만 삭제 가능
- 삭제 시 해당 룸의 총 주문액 재계산
- 삭제된 주문은 DB에서 물리 삭제

## BR-ROOM: 룸 세션 규칙

### BR-ROOM-01: 세션 시작
- 해당 룸에서 첫 주문이 들어올 때 자동 시작
- 기본 이용 시간: 2시간
- expiresAt = startedAt + 2시간

### BR-ROOM-02: 세션 만료 알림
- 만료 10분 전 고객에게 알림
- 만료 시 추가 알림 (합석 연장이 아닌 경우 관리자가 수동 처리)

### BR-ROOM-03: 이용 완료 (관리자)
- 관리자가 이용 완료 버튼 클릭
- 해당 세션의 모든 주문 → OrderHistory로 이동
- 룸 세션 비활성화 (active = false, completedAt 기록)
- 룸의 현재 주문 목록 및 총 주문액 리셋
- 다음 고객은 새 세션으로 시작

## BR-MENU: 메뉴 관리 규칙

### BR-MENU-01: 메뉴 등록
- 필수: 메뉴명, 가격, 카테고리
- 가격: 0 이상 정수
- 이미지: 서버 로컬 업로드 (선택)
- 주류 병/잔은 별도 메뉴로 등록

### BR-MENU-02: 메뉴 순서
- displayOrder 값으로 정렬
- 같은 카테고리 내에서 순서 조정

## BR-CALL: 직원 호출 규칙

### BR-CALL-01: 호출 생성
- 사유 선택 필수 (물, 잔, 휴지, 젓가락, 숟가락, 에어컨, 앞접시 등)
- 호출 시 SSE로 관리자에게 실시간 알림
- 동일 룸에서 중복 호출 가능

### BR-CALL-02: 호출 처리
- 관리자가 처리 완료 표시
- handled = true, handledAt 기록
