# User Stories

## Priority Legend (MoSCoW)
- **Must**: MVP 필수
- **Should**: MVP에 포함되면 좋음
- **Could**: 향후 추가 가능
- **Won't**: 이번 범위 제외

---

## Epic 1: 핵심 주문 (Core Order)

### US-01: 룸 태블릿 자동 로그인 [Must]
**As a** 고객, **I want** 룸 태블릿에서 별도 로그인 없이 바로 주문할 수 있기를, **So that** 대기 시간 없이 즉시 이용할 수 있다.

**Acceptance Criteria:**
- 관리자가 초기 설정(매장ID, 룸번호, 비밀번호) 완료 시 자동 로그인 활성화
- 브라우저 새로고침 시에도 로그인 유지
- 16시간 세션 만료 후 재로그인 필요

### US-02: 메뉴 조회 [Must]
**As a** 고객, **I want** 카테고리별로 메뉴를 탐색할 수 있기를, **So that** 원하는 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria:**
- 좌측 사이드바에서 카테고리 선택 시 해당 메뉴 표시
- 카드 그리드(2열)로 메뉴명, 가격, 이미지 표시
- 메뉴 클릭 시 상세 정보(설명, 이미지) 확인 가능

### US-03: 장바구니 관리 [Must]
**As a** 고객, **I want** 주문 전 메뉴를 장바구니에 담고 수정할 수 있기를, **So that** 최종 주문 전에 내역을 확인하고 조정할 수 있다.

**Acceptance Criteria:**
- 메뉴 추가/삭제, 수량 증감 가능
- 총 금액 실시간 계산
- 페이지 새로고침 시에도 장바구니 유지 (로컬 저장)

### US-04: 주문 생성 [Must]
**As a** 고객, **I want** 장바구니의 메뉴를 주문할 수 있기를, **So that** 주방에 주문이 전달된다.

**Acceptance Criteria:**
- 주문 확정 시 주문 번호 표시
- 5초 후 메뉴 화면으로 자동 리다이렉트
- 주문 실패 시 에러 메시지 표시, 장바구니 유지

### US-05: 주문 내역 조회 [Must]
**As a** 고객, **I want** 현재 세션의 주문 내역을 확인할 수 있기를, **So that** 지금까지 주문한 내역과 상태를 파악할 수 있다.

**Acceptance Criteria:**
- 현재 룸 세션 주문만 표시
- 주문별 번호, 시각, 메뉴/수량, 금액, 상태 표시
- 주문 시간 순 정렬

## Epic 2: 소셜/헌팅 (Social & Hunting)

### US-06: 음료/안주 보내기 [Must]
**As a** 고객, **I want** 다른 룸에 음료나 안주를 보낼 수 있기를, **So that** 관심 있는 룸에 호감을 표현할 수 있다.

**Acceptance Criteria:**
- 메뉴 선택 + 받는 룸 번호 입력
- 받는 룸에 수락/거절 알림 표시
- 수락 시 주방에 주문 전달 (보내는 사람의 룸이 과금)
- 거절 시 보내는 룸에 거절 알림

### US-07: 메시지 보내기 [Should]
**As a** 고객, **I want** 다른 룸에 메시지를 보낼 수 있기를, **So that** 직접 가지 않고도 소통할 수 있다.

**Acceptance Criteria:**
- 특정 룸 번호 지정 1:1 메시지 전송
- 받는 룸에 실시간 메시지 알림
- 메시지 수신 시 발신 룸 번호 표시

### US-08: 사진 촬영/보내기 [Should]
**As a** 고객, **I want** 태블릿으로 사진을 찍어 다른 룸에 보낼 수 있기를, **So that** 시각적으로 소통할 수 있다.

**Acceptance Criteria:**
- 태블릿 카메라로 즉석 촬영
- 특정 룸에 1:1 전송
- 수신 룸에서 사진 확인 가능

### US-09: 합석 요청 [Must]
**As a** 고객, **I want** 다른 룸에 합석을 요청할 수 있기를, **So that** 새로운 사람들과 함께 즐길 수 있다.

**Acceptance Criteria:**
- 합석 요청 시 받는 룸에 수락/거절 알림
- 수락 시 합석 성립, 기본 30분 무료 연장
- 거절 시 요청 룸에 거절 알림

### US-10: 합석 시간 연장 [Must]
**As a** 고객, **I want** 합석 후 시간을 연장할 수 있기를, **So that** 즐거운 시간을 더 보낼 수 있다.

**Acceptance Criteria:**
- 최초 1회 기본 30분 무료 연장 후 30분마다 재알림
- 연장 시 최소 술 1병 주문 필수
- 주문 없이 연장 불가

### US-11: 콘텐츠 신고 [Should]
**As a** 고객, **I want** 부적절한 사진/메시지를 신고할 수 있기를, **So that** 불쾌한 경험으로부터 보호받을 수 있다.

**Acceptance Criteria:**
- 수신한 사진/메시지에 신고 버튼
- 신고 시 관리자에게 알림 전달
- 신고 접수 확인 피드백

## Epic 3: 엔터테인먼트 (Entertainment)

### US-12: 합석 미니게임 [Must]
**As a** 고객, **I want** 합석 성공 시 미니게임을 할 수 있기를, **So that** 어색한 분위기를 깨고 자연스럽게 대화할 수 있다.

**Acceptance Criteria:**
- 합석 성공 시 미니게임 메뉴 자동 노출 (이스터에그)
- 게임 종류 선택 가능 (아이스브레이킹/룰렛/밸런스 등)
- 양쪽 룸 태블릿에서 동시 참여 가능

### US-13: 아이스브레이킹 질문 게임 [Must]
**As a** 고객, **I want** 랜덤 질문으로 대화를 시작할 수 있기를, **So that** 처음 만난 사람과 자연스럽게 이야기할 수 있다.

**Acceptance Criteria:**
- 랜덤 질문 출제
- 다음 질문 넘기기 가능
- 다양한 질문 카테고리 (가벼운/재미있는/깊은 대화)

### US-14: 룰렛/사다리/벌칙 게임 [Should]
**As a** 고객, **I want** 룰렛이나 사다리타기 같은 미니게임을 할 수 있기를, **So that** 재미있는 벌칙/미션으로 분위기를 띄울 수 있다.

**Acceptance Criteria:**
- 룰렛, 사다리타기, 랜덤 벌칙 중 선택
- 결과 화면 표시
- 참여 인원 수 설정 가능

### US-15: 밸런스/진실거짓 게임 [Should]
**As a** 고객, **I want** 밸런스 게임이나 진실 or 거짓 게임을 할 수 있기를, **So that** 서로에 대해 알아갈 수 있다.

**Acceptance Criteria:**
- 랜덤 밸런스 질문 또는 진실/거짓 주제 출제
- 선택 결과 표시
- 다음 문제 넘기기 가능

### US-16: 이야기 주제 랜덤 생성 [Should]
**As a** 고객, **I want** 대화 주제를 랜덤으로 추천받을 수 있기를, **So that** 대화가 끊기지 않고 이어갈 수 있다.

**Acceptance Criteria:**
- 랜덤 대화 주제 생성
- 주제 카테고리 선택 가능 (가벼운/깊은/재미있는)
- 다음 주제 넘기기 가능

## Epic 4: 유틸리티 (Utility)

### US-17: 직원 호출 [Must]
**As a** 고객, **I want** 태블릿에서 직원을 호출할 수 있기를, **So that** 필요한 물품을 빠르게 요청할 수 있다.

**Acceptance Criteria:**
- 호출 사유 선택 (물, 잔, 휴지, 젓가락, 숟가락, 에어컨, 앞접시 등)
- 호출 시 관리자 화면에 실시간 알림
- 호출 접수 확인 피드백

### US-18: 이용 시간 확인 [Must]
**As a** 고객, **I want** 남은 이용 시간을 확인할 수 있기를, **So that** 시간 관리를 할 수 있다.

**Acceptance Criteria:**
- 기본 2시간 이용 시간 표시
- 남은 시간 실시간 카운트다운
- 시간 초과 시 알림 표시

## Epic 5: 관리자 (Admin)

### US-19: 매장 로그인 [Must]
**As a** 관리자, **I want** 매장 관리 시스템에 로그인할 수 있기를, **So that** 매장을 운영할 수 있다.

**Acceptance Criteria:**
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 기반 16시간 세션 유지
- 로그인 시도 제한

### US-20: 실시간 주문 모니터링 [Must]
**As a** 관리자, **I want** 들어오는 주문을 실시간으로 확인할 수 있기를, **So that** 빠르게 주문을 처리할 수 있다.

**Acceptance Criteria:**
- SSE 기반 실시간 업데이트 (2초 이내)
- 룸별 카드 그리드에 총 주문액, 최신 주문 표시
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조

### US-21: 룸 관리 [Must]
**As a** 관리자, **I want** 룸의 상태를 관리할 수 있기를, **So that** 효율적으로 매장을 운영할 수 있다.

**Acceptance Criteria:**
- 룸 초기 설정 (번호, 비밀번호)
- 이용 완료 처리 (세션 종료 → 과거 이력 이동 → 리셋)
- 과거 주문 내역 조회 (날짜 필터링)

### US-22: 주문 삭제 [Must]
**As a** 관리자, **I want** 잘못된 주문을 삭제할 수 있기를, **So that** 주문 오류를 정정할 수 있다.

**Acceptance Criteria:**
- 삭제 확인 팝업 표시
- 삭제 후 총 주문액 자동 재계산
- 성공/실패 피드백

### US-23: 메뉴 관리 [Must]
**As a** 관리자, **I want** 메뉴를 등록/수정/삭제할 수 있기를, **So that** 메뉴를 최신 상태로 유지할 수 있다.

**Acceptance Criteria:**
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지)
- 이미지 서버 업로드
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

### US-24: 매장 관리 [Must]
**As a** 관리자, **I want** 매장 정보를 관리할 수 있기를, **So that** 다중 매장을 운영할 수 있다.

**Acceptance Criteria:**
- 매장 등록/수정/삭제
- 매장별 정보 관리
- 다중 매장 지원

### US-25: 직원 호출 관리 [Must]
**As a** 관리자, **I want** 고객의 직원 호출을 실시간으로 확인할 수 있기를, **So that** 빠르게 대응할 수 있다.

**Acceptance Criteria:**
- SSE 기반 실시간 호출 알림
- 호출 사유 및 룸 번호 표시
- 처리 완료 표시

### US-26: 신고 관리 [Should]
**As a** 관리자, **I want** 고객의 신고를 확인하고 처리할 수 있기를, **So that** 매장 내 부적절한 행위를 관리할 수 있다.

**Acceptance Criteria:**
- 신고 접수 목록 확인
- 부적절 콘텐츠 삭제 처리
- 처리 완료 표시
