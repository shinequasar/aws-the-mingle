# 룸 헌팅포차 테이블오더 서비스 요구사항

## Intent Analysis Summary
- **User Request**: 룸 헌팅포차 전문 테이블오더 서비스 신규 구축
- **Request Type**: New Project (Greenfield)
- **Scope**: Multiple Components (고객용 UI, 관리자용 UI, 백엔드 API, 데이터베이스)
- **Complexity**: Complex (실시간 SSE, JWT 인증, 세션 관리, 파일 업로드, 다중 매장, 합석 시스템, 룸 간 소통, 미니게임)

## 1. Functional Requirements

### 1.1 고객용 기능 (Customer App)

#### FR-C01: 룸 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 식별자, 룸 번호, 룸 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지
- 룸 번호 = 테이블 번호 (1:1 매핑)

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리: MZ 감성 메뉴 구성 (간단한 안주, 기발한 세트메뉴, 폭탄주, 트렌디한 술)
- 좌측 사이드바 카테고리 네비게이션
- 카드 그리드 (2열) 레이아웃
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- 주류는 병/잔 별도 메뉴로 등록
- 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 성공 시: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트 → 장바구니 비우기
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 룸 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 룸 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)

#### FR-C06: 룸 간 소통
- **음료/안주 보내기**: 메뉴 선택 + 받는 룸 번호 입력 → 받는 룸에 수락/거절 알림 → 수락 시 주방 전달 (보내는 룸에 과금)
- **메시지 보내기**: 특정 룸 번호 지정 1:1 메시지
- **사진 보내기**: 태블릿 카메라로 즉석 촬영 → 특정 룸에 1:1 전송
- **신고 기능**: 부적절한 사진/메시지 수신 시 신고 → 관리자 확인

#### FR-C07: 합석 시스템
- 룸 A가 룸 B에 합석 요청 전송
- 룸 B가 수락/거절
- 수락 시 합석 성립
- **합석 시간 연장 규칙**:
  - 합석 성립 시 최초 1회 기본 무료 연장 30분
  - 30분 경과 후 재알림 (연장 여부 확인)
  - 이후 30분마다 반복 재알림
  - 연장 시 최소 술 1병 주문 필수 (주문 없이 연장 불가)
- 합석 성공 시 미니게임 자동 실행

#### FR-C08: 미니게임 (합석 성공 시 + 레크리에이션 메뉴)
- **아이스브레이킹 질문 게임**: 랜덤 질문 출제 → 돌아가며 답변
- **미니게임**: 룰렛, 사다리타기, 랜덤 벌칙 등
- **대화형 게임**: 진실 or 거짓, 밸런스 게임
- **이야기 주제 랜덤 생성**: 대화 소재 제공
- 합석 성공 시 자동으로 미니게임 메뉴 노출 (이스터에그)

#### FR-C09: 직원 호출
- 호출 사유 선택: 물, 잔, 휴지, 젓가락, 숟가락, 에어컨, 앞접시 등
- 호출 시 관리자 화면에 알림

#### FR-C10: 이용 시간 관리
- 기본 이용 시간 2시간
- 남은 시간 표시
- 시간 초과 알림
- 합석 성립 시 무료 연장 (기본 30분 + 이후 30분마다 재알림, 연장 시 술 1병 주문 필수)

### 1.2 관리자용 기능 (Admin App)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트 (2초 이내)
- 룸별 카드 형태 그리드 레이아웃 (심플 테이블/리스트 기반)
- 각 룸 카드: 총 주문액, 최신 주문 미리보기, 남은 이용 시간
- 주문 카드 클릭 시 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조

#### FR-A03: 룸 관리
- 룸 초기 설정 (번호, 비밀번호, 세션 생성)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- 룸 이용 완료 (세션 종료 → 주문 내역 과거 이력 이동 → 리셋)
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)
- 합석 현황 모니터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지)
- 이미지: 서버 직접 업로드 (로컬 파일 저장)
- 메뉴 노출 순서 조정
- 주류 병/잔 별도 메뉴 등록
- 필수 필드 및 가격 범위 검증

#### FR-A05: 매장 관리
- 다중 매장 지원
- 매장 등록/관리 기능

#### FR-A06: 직원 호출 관리
- 실시간 호출 알림 수신 (SSE)
- 호출 사유 및 룸 번호 표시
- 호출 처리 완료 표시

#### FR-A07: 신고 관리
- 사진/메시지 신고 접수 확인
- 부적절한 콘텐츠 삭제 처리

### 1.3 UI/UX 요구사항

#### 고객용 앱
- 디자인 톤: 모던/미니멀 (화이트, 그레이 기반)
- 메뉴 레이아웃: 카드 그리드 (2열), 이미지 중심
- 카테고리: 좌측 사이드바 (세로 고정)
- 터치 친화적 인터페이스
- 브랜딩: 로고 + 매장명 표시

#### 관리자용 앱
- 심플한 테이블/리스트 기반 (기능 중심)
- 별도 독립 웹 애플리케이션

## 2. Non-Functional Requirements

### NFR-01: 성능
- SSE 기반 실시간 주문/알림 표시 2초 이내
- 메뉴 조회 응답 시간 1초 이내

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 시각적 계층 구조
- 장바구니 로컬 저장 (새로고침 유지)

### NFR-03: 보안
- JWT 토큰 기반 인증
- bcrypt 비밀번호 해싱
- 로그인 시도 제한
- 16시간 세션 만료

### NFR-04: 데이터 무결성
- 주문 생성 시 트랜잭션 처리
- 세션 기반 주문 그룹화
- 이용 완료 시 과거 이력 보존

## 3. 제외 범위 (Out of Scope)
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)
